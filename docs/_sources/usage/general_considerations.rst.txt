.. _general_considerations:

======================
General considerations
======================

Naming conventions
------------------
For the sake of clarity in this multi-language documentation, the following naming conventions have been adopted.

Classes
=======
- Class names are written in Pascal case. Example: ``License``.
- Class method names are written in Camel case. Example: ``activateSerialKey``
- Class property names are written in Camel case. Example: ``Image.pixelFormat``
  

Enumerations
============
- Enumerations are always written in Camel case. Example : ``LicenseError.invalidLicense``.

The field name should be formatted according to the programming language. For example:

.. tab-set::

    .. tab-item:: C
        :sync: c

        The field name is format as follow: id3<enumeration_name>_<field_name>.

        .. code-block:: C

            id3LicenseError_InvalidLicense

    .. tab-item:: Dart
        :sync: dart

        .. code-block:: dart

            LicenseError.invalidLicense

    .. tab-item:: C#
        :sync: csharp

        .. code-block:: c#

            LicenseError.InvalidLicense

    .. tab-item:: Java
        :sync: java
   
        .. code-block:: java
        
            LicenseError.invalidLicense

    .. tab-item:: Python
        :sync: python

        .. code-block:: python

            LicenseError.invalidLicense

    .. tab-item:: Swift
        :sync: swift

        .. code-block:: swift

            LicenseError.InvalidLicense
