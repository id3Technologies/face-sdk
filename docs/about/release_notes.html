Release notes#

Version 9.9.1 (2023-10-18)#

  • Changed default PortraitProcessor.mouthOpenThreshold value from 5 to 35.

  • Fixed PortraitProcessor.detectRedEyes method.

  • Adds documentation and sample code for the PortraitProcessor module.

  • Skips color PAD in PortraitProcessor if model is not loaded.

Version 9.9.0 (2023-10-13)#

New features

  • Improved presentation attack detection in PortraitProcessor module.

API changes

  • PadStatus enumeration added.

  • Portrait.isPresentationAttack replaced by Portrait.padStatus.

  • PortraitQualityAttributes member names changed for clarity.

Version 9.8.0 (2023-09-27)#

New features

  • New unified multi-language documentation.

  • New model for face PAD: FaceAttackSupportDetector3A.

  • Inference engine updated.

API changes

  • FaceAnalyser.applySegmentationMask method renamed FaceAnalyser.applyMask and now takes 3 parameters to set the background color.

  • FaceAnalyser.applyAlphaMask method added => returns a BGRA image with transparent background.

  • FaceAnalyser.threadCount property added.

  • License module renamed FaceLicense.

  • FaceLibrary.checkLicense methods moved to FaceLicense module.

  • FaceLibrary.getVersionEx method added.

  • FaceTemplateBufferType enumeration removed.

  • Some enumerations renamed in Dart (ex: FaceTemplateFormat) and Java (ex: PixelFormat).

  • New Swift API.

  • Updated Python package API :

    • Camel case for methods and attributes, upper case for enumerations.

    • Renamed package to lower case.

  • New wrapper for Python

  • Added build for NXP iMX (Edge Edition)

  • PortraitProcessor module:

    • Improved portrait quality computation - Bug fixes

  • FaceAnalyser module:

    • Fix under-exposure and over-exposure default sensibility values

Version 9.5.0 (2023-03-21)#

  • Added Face Detector 4A

  • Added new Portrait Processor module used to facilitate the creation of portraits in various scenarios such as enrolment of face verification. The module performs the following tasks:

    • Face detection and tracking with prediction models

    • Landmark detection

    • Head pose estimation

    • Subject’s position verification

    • Template encoding and updating

    • Additional optional computations (age estimation, background uniformity computation, emotion detection, eye gaze detection, eye opening detection, face mask detection, gender estimation, glasses detection, hat detection, make-up detection, mouth opening estimation, occlusion detection, photographic attributes computation, presentation attack detection, red-eye detection, smile detection)

    • Verification of compliance on many quality attributes

    • Unified quality score computation

  • Fixed left/right eye in occlusions

  • Added method to update a FaceTemplate

  • Added FaceAnalyzer.computePoseWithLandmarks() method for more accurate pose estimation.

  • Added DetectedFace.getExpandedBounds() method.
